this document serves as a guide on how to contribute to the project.
below are three scenarios when contributing.

## You want to make a change in the codebase and there is not issue related to it

- open a new issue. make sure to follow how to create an issue guidelines
- create a new branch
- make your changes
- commit your changes
- push your changes 
- open a pull reqest.

## You see an open issue without an assigned developer

- assign yourself to the issue.
- create a new branch
- make your changes
- commit your changes
- push your changes 
- open a pull reqest.

## you are assigned to an issue

- understand the requirements
- create a new branch
- make your changes
- commit your changes
- push your changes 
- open a pull reqest.

# footnote:

for all scenarios above, you must always adhere to our collaboration guidelines.

- 701 - opening an issue
- 702 - submitting a pull request
- 703 - naming your branches
- 704 - commit conventions
